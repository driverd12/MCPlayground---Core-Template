# MCPlayground Gemini Instructions

Use the MCP server as the primary control plane for this repository.

Core workflow:
- Start with `operator.brief` when you need the current bounded objective.
- Use `office.snapshot` or `kernel.summary` to inspect live system state.
- Route new operator objectives through `autonomy.ide_ingress`.
- Prefer the local-first council before escalating to hosted providers.

Local-first council:
- `implementation-director`
- `research-director`
- `verification-director`
- `local-imprint`

Execution rules:
- Keep work bounded.
- State success criteria, evidence, and rollback clearly.
- Do not invent a second ingress path.
- Do not perform recursive self-improvement.
- If external auth or human approval is required, record the blocker and stop.

Reliability rules:
- Use `autonomy.maintain` for bounded upkeep.
- Use `provider.bridge` to inspect provider connectivity.
- Prefer existing MCP tools over ad hoc shell workflows when the tool already exists.
